# e-dialect 社区健康配置

欢迎来到 e-dialect 开源社区！👋

本仓库 (`e-dialect/.github`) 是 e-dialect 组织的 **社区健康配置仓库**。它包含了适用于组织下所有成员仓库的默认文件和社区治理文档。

## 📂 仓库内容

本仓库主要包含以下核心文件，它们为 e-dialect 社区的所有项目提供统一的指引和规范：

| 文件 | 说明 |
| :--- | :--- |
| [`CONTRIBUTING.md`](./CONTRIBUTING.md) | **贡献指南**：详述了如何为 e-dialect 项目贡献代码，包括 Issue 认领、Fork/PR 流程、Git 提交规范、分支管理策略等。 |
| [`CODE_OF_CONDUCT.md`](./CODE_OF_CONDUCT.md) | **行为准则**：定义了社区成员应遵守的行为规范，致力于营造一个开放、友好、包容的社区环境。 |
| [`ICLA.md`](./ICLA.md) | **个人贡献者许可协议**：个人贡献者在提交代码时需同意的协议，保障项目知识产权清晰。 |
| [`CCLA.md`](./CCLA.md) | **企业贡献者许可协议**：企业或机构作为贡献者时需签署的协议。 |

> **💡 提示**：GitHub 会自动将本仓库中的 `CONTRIBUTING.md` 和 `CODE_OF_CONDUCT.md` 等文件作为组织下所有仓库的默认文件，除非某个仓库自己定义了同名文件。

## 🚀 如何参与贡献

我们非常欢迎社区伙伴参与到 e-dialect 的建设中来！无论你是想修复一个错别字、增加一个新功能，还是完善文档，你的帮助都至关重要。

在开始之前，请务必仔细阅读我们的 **[贡献指南 (CONTRIBUTING.md)](./CONTRIBUTING.md)**。

### 快速参与步骤

1. **寻找任务**：前往感兴趣的项目仓库，查看标记为 `good first issue` 或 `help wanted` 的 Issues。
2. **认领 Issue**：在 Issue 下方留言或自行 Assign，表明你正在处理该问题。
3. **签署协议**：
    *   **个人贡献者**：提交 PR 即视为同意 [ICLA](./ICLA.md)。
    *   **企业贡献者**：请先让贵公司签署 [CCLA](./CCLA.md)。
4. **提交代码**：Fork 仓库 -> 创建分支 -> 提交代码 -> 发起 Pull Request。
5. **代码评审**：等待社区成员进行 Code Review，并根据反馈进行修改。
